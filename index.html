About Me

I'm a software engineer who genuinely enjoys building things. Not just shipping features — understanding systems end to end, from the database query plan to the user experience. Five years across fintech, gaming, and enterprise AI, and the pattern is the same: I see a problem, I build something to solve it, and I learn something new along the way.

The common thread: I gravitate toward hard problems where distributed systems meet emerging technology.

When a new tool drops — a new LLM, an agent framework, a dev tool — I don't just read the blog post. I build something with it. That's where the experiments below come from. Some turned into product concepts, some into architecture patterns I now use at work, and some were just fun to build.
